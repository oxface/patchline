# Docs

This folder holds durable project documentation.

Use this folder for decisions, behavior specs, roadmap, architecture notes, learning logs, and references that multiple parts of the repository need to share.

Folder-level `README.md` and `AGENTS.md` files should point back here instead of duplicating long-lived context.

## Key Areas

- `decisions/`: durable architecture and product decisions.
- `behaviors/`: concrete workflow behavior specifications.
